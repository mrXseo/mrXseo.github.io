

***

**Gunverse** — фреймворк для создания высокопроизводительных игровых сущностей в Godot Engine. Спроектирован так, чтобы дизайнеры и программисты работали в одной среде без конфликтов.

**Киллер-фичи для команды:**
* Дизайнер создаёт целые классы пуль без единой строки кода — всё через Inspector.
* Полная интеграция в родной интерфейс Godot, никаких внешних редакторов.
* Программист добавляет новое поведение как плагин (один `.gd`-файл) — система сама подхватывает его и расширяет инструментарий дизайнера.

**Что под капотом:**
* **ECS-подобное ядро с Data-Oriented Design:** данные лежат в кэш-дружественных массивах, поведение — композиция независимых Callable.
* **Zero-If в рантайме:** все условия разрешаются на этапе сборки; во время игры — только прямые вызовы.
* **Command Buffer для безопасности:** изменение состава процессов во время кадра не ломает итерацию — изменения применяются атомарно на следующем кадре.
* **Реестр сборки:** детальные логи происхождения каждого слота (provenance) для отладки.

**Реальный пример:**
Стартовый набор из 3 плагинов даёт дизайнеру настройку кривых траекторий для движения по прямой. Добавив **1 плагин за 1 час**, программист открывает дизайнеру:
* любые траектории в любом направлении,
* зависимость скорости от урона и наоборот,
* отскоки, пробивание, эффекты попаданий,
* самонаведение на разные цели.

При этом код ядра не меняется.

**Производительность (тесты на AMD Ryzen 5 3550G, встроенная графика):**
* до 2500 сущностей с одним процессом при 60 FPS,
* до 900 сущностей с четырьмя процессами при 60 FPS.
Это близко к пределу возможностей Godot без замены рантайма на нативный код.

**Надёжность:** чёткая архитектура и fail-fast на этапе сборки исключают баги на стыках процессов — некорректная конфигурация не долетает до сцены.

**Затрачено:** 57 часов на проектирование, реализацию и тестирование.
