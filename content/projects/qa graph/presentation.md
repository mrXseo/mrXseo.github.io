**QA Graph Migrator** — конвейер для автоматической очистки и унификации тестовых данных. Сложные графы проверяются простыми правилами, а тестовый сьют получает только чистые данные.

**Киллер-фичи для команды:**
- Тестировщик получает «чистое» пространство данных одной функцией — никакой ручной правки JSON или базы.
- Добавление нового продукта — два файла-стандарта: интерпретация + валидация. Переписывать ядро не нужно.
- Лид, который не дружит с классами, может расширить проверки по шаблону из README за полчаса.
- Система отлавливает невалидные значения на входе и заменяет их на документированные умолчания — баги форматов больше не маскируются под баги продукта.

**Что под капотом:**
- **Графовое ядро `HierarchicalEntity` с монадической рекурсией (State Monad).** Узлы — устройства, компоненты, входы/выходы; каскадные задачи обходят связи автоматически. Эхо‑резистентность исключает зацикливание обходов.
- **Двухфазный конвейер:** Интерпретация (нормализация полей по документации, например версия «2,01» → 201) → Валидация (жёсткая проверка по типам и диапазонам). Фазы настраиваются раздельно.
- **Мигратор с авторазрешением зависимостей:** вычисляет порядок применения правил, не требуя ручной сортировки полей. При неразрешимых зависимостях выдаёт читаемый лог.

**Измеренное влияние:**
- **×24 ускорение** прогона тестов при работе с БД — однократная загрузка и кэширование таблиц типов (`DBTable`) вместо тысяч запросов.
- **–76% сквозных ошибок** (с ~30% до <10% падений из-за несовпадения форматов).
- **< 1 часа** на добавление поддержки нового типа устройств (против нескольких дней ручной адаптации).
- **–60% объёма кода** в тестовых сценариях благодаря фасаду сравнения.

**Надёжность и расширяемость:**
- Fail‑fast на этапе сборки: неподдерживаемый тип источника или несовпадение полей интерпретации и валидации ловятся сразу.
- Модульная архитектура: ядро, миграция и инструменты разделены. Утилиты (поиск узлов, компаратор пространств) переиспользуются без дублирования.
- Устойчивость к изменениям продуктов: бизнес‑логика тестов не содержит кода нормализации и не сломается при смене структуры конфигов.

**Стек:** Python, PostgreSQL, pytest.

**Паттерны:** State Monad, Interpreter, Теория категорий (формализация), Fail‑fast.

**Контекст:** Реализовано с нуля за время стажировки в отделе QA. Самостоятельно спроектирована архитектура, графовое ядро, миграционный движок и библиотека инструментов для команды.

**Затрачено:** 83 часа на проектирование, реализацию ядра, мигратора, загрузчиков, стандартов для двух реальных систем и набора инструментов.
